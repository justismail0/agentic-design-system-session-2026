Agent layer6 min
Three tools, one agent

Three tools, one agent.

Don't confuse the entry point with the capability. Same agent — three places it can live. We're using the Desktop app today.

CLI

claude in a terminal. The agent runs locally, reads your project, and writes back changes you can diff and approve before anything lands.

Reach for it when you want scriptability — when you'd rather chain a session into your build pipeline than click through a window.

The wrong choice if you don't live in terminals. The Desktop and IDE versions exist for exactly that reason.

Desktop App

A standalone Claude app for Mac and Windows. Same agent, same file access — but inside a chat window. Closer to ChatGPT than to a terminal.

Reach for it when the work is conversational: a design review, exploring an idea, multi-step thinking where you want to see the back-and-forth as a thread. It's also the surface for today, because the room can follow what's on screen.

Not the place for scripts or anything that runs unattended — there's no headless mode.

IDE plugin

Claude inside the file you're already in. ⌘K opens an inline prompt against the current selection — no app switch, no copy-paste.

The right tool when the question is local: tweak this rule, fix this binding, rewrite this function. The agent operates at the cursor, not at the project.

It can't see beyond the file. For project-wide reasoning, switch to the CLI or the Desktop app.

# CLAUDE.md

Project context for Claude Code working in this design-system repo.

## 1 · Project identity

Token-driven design system. Figma is the source of truth for every
token. The local tokens.json is the consolidated DTCG export — the
same data, in a file the agent can read alongside the Figma file.

- Figma file: vdVmrWthRNIyM1lf2XThdZ
- Welcome screen: node 1:51
- Foundations page: node 2:82
  - Primitive colors planning canvas: node 2:3389
  - Semantic colors planning canvas: node 2:3856

## 2 · Architecture summary

Primitive → Semantic → Component

Three tiers. Components reference semantic tokens only — never primitives
directly. Per-component bindings live alongside the component, not in a
shared collection. The split files in foundations/ and tokens/
mirror sections of tokens.json; the consolidated file is the source.

## 3 · Reference tables

| Collection                | Modes              | Role |
|---------------------------|--------------------|---|
| Primitive: Colors     | 1                  | Raw palette scales (yellow / teal / green / red / orange / blue / neutral × 50–900, base) |
| Primitive: Typography | 1                  | font-family, font-weight (STRING); font-size, line-height (FLOAT) |
| Primitive: Size       | 1                  | 4px-grid scale (0–999) |
| Semantic: Colors      | Light, Dark        | Property/role/state aliases to primitives |
| Semantic: Typography  | 1                  | Aliases to primitive typography |
| Semantic: Size        | 1                  | spacing, border-radius, border-width, icon, component-height |

### Scopes by token type

| Token area              | Scopes |
|-------------------------|---|
| Color: background/*     | FRAME_FILL, SHAPE_FILL |
| Color: text/*           | TEXT_FILL |
| Color: icon/*           | SHAPE_FILL, TEXT_FILL |
| Color: border/*         | STROKE_COLOR |
| Primitive colors        | ALL_FILLS, STROKE_COLOR, EFFECT_COLOR |
| Dimensions              | GAP, WIDTH_HEIGHT (spacing), CORNER_RADIUS (radius), STROKE_FLOAT (border-width), FONT_SIZE, LINE_HEIGHT, FONT_FAMILY, FONT_STYLE |

## 4 · Conventions

- Semantic colors: {property}/{role}/{state} (e.g.
  background/brand/default-hover, text/primary/disabled). Split
  composite state keys on the first hyphen only so solid-hover
  stays as one state.
- Everything else: slash-grouped paths (border-radius/full,
  font-size/headline-xl).
- Primitives: {group}/{step} (yellow/500, neutral/100).
- Per-component files: one folder per component with three files
  ({name}.tokens.json authored, {name}.html + {name}.md generated
  by the component-documentation skill; judgment sections preserved
  across regen).

## 5 · Critical rules

1. Always set variable.scopes explicitly. The ALL_SCOPES default
   pollutes pickers.
2. tokens.json is the source — never hand-edit the split files in
   foundations/ or tokens/. Round-trip changes through Figma.
3. Tier reference rule. Molecule tokens may reference atom tokens via
   $alias paths. Never the other way around.
4. Patterns are MD-only. No tokens.json or preview.
5. State tokens come from semantic -hover / -disabled tokens —
   don't invent new component-level state tokens.
6. Active state has no semantic token. CSS filter: brightness(0.95)
   is the stopgap. Document it in the component MD's Known issues.

## 6 · Workflow patterns

- Required preflight. Always load figma:figma-use before any
  use_figma call. Pass skillNames: "figma-use" on every invocation.
- Inspect first, write second. Always run a read-only use_figma
  (list collections, list component names) before creating anything.
- Small atomic steps. Pattern: inspect → primitives → semantics →
  components → validate.
- Page-per-component. Every component build creates a new Figma page
  named after the component. Place the component set on that page.
- Cross-collection aliasing order. Primitives must exist before
  semantics alias them; semantics before components.
- Return output via figma return values. console.log is invisible.
  Every script that mutates nodes must return affected IDs:
  return { createdNodeIds: [...], mutatedNodeIds: [...] };

## 7 · File references

Top-level keys in tokens.json are collection slugs:
primitive-colors, primitive-size, primitive-typography,
semantic-colors, semantic-size, semantic-typography.

Each collection has metadata ($collection, $slug, $modes,
$defaultMode, $description) and a tree of tokens. Leaf tokens use
$type (COLOR/FLOAT/STRING), $scopes, $values keyed by mode name.
Aliases use {"$alias": "{collection-slug.path.to.var}"}.

*.tokens.json top-level keys: $source, $component,
$figmaNodeId, $description, $variants, $properties,
$anatomy, $layout, then per-variant token tables.

## 8 · Discovered rules ● appended live

Append new rules here the moment they're learned. Each rule should
stand alone with enough context that a fresh session can apply it.

### Fonts

- Inter — Semi-bold is "Semi Bold" (with a space). Same for
  "Extra Bold". Using "SemiBold" fails loadFontAsync.
- SF Pro — Semi-bold is "Semibold" (one word). Different from Inter.
- When loadFontAsync throws, call figma.listAvailableFontsAsync()
  to discover the exact style string rather than guessing.

### Figma variable API

- hiddenFromPublishing on variables and collections throws "Node not
  found". Toggle manually in the Local Variables panel.
- figma.notify() throws "not implemented" — never use it. Return via
  return.
- figma.currentPage = page throws "not supported". Use
  await figma.setCurrentPageAsync(page).
- getPluginData / setPluginData are not supported. Use
  getSharedPluginData / setSharedPluginData with a stable 3+ char
  namespace.
- Setting layoutSizingHorizontal/Vertical = 'FILL' must happen
  after parent.appendChild(child). Setting before throws.
- setBoundVariableForPaint returns a new paint — capture and reassign,
  don't assume in-place mutation.
- addComponentProperty(name, "INSTANCE_SWAP", defaultValue) —
  defaultValue must be the node ID, NOT component.key.

### Runtime limits (Figma MCP)

- Page context resets between use_figma calls — always
  await figma.setCurrentPageAsync(page) at the start.
- Scripts are atomic — on error, no partial state is written; safe to
  fix and retry.

### Semantic naming

- Split composite {role}-{state} keys on first hyphen only.
  brand-default-hover → brand/default-hover, not
  brand/default/hover. Keeps default-hover and solid-hover
  readable as one state unit.
